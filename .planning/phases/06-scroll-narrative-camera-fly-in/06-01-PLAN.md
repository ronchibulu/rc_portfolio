# Plan 06-01: GSAP ScrollTrigger Camera Fly-In Hook

**Phase:** 6 — Scroll Narrative + Camera Fly-In  
**Requirements:** SCROLL-001, SCROLL-002, SCROLL-004, SCROLL-005, SCROLL-006, SCROLL-007

## Goal

Wire GSAP ScrollTrigger inside `GameSetupScene.tsx` to drive the camera from its reference angle toward the monitor face. GSAP writes to a plain JS progress ref; `useFrame` reads it and applies the interpolated position/target to the Three.js camera. `$scrollProgress` nanostore updated each frame. `invalidate()` called from GSAP ticker during active scroll.

## Tasks

### Task 1 — Register GSAP plugins at module level in GameSetupScene.tsx

At the top of `src/components/canvas/GameSetupScene.tsx`, add:

```ts
import { gsap } from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';
import { useGSAP } from '@gsap/react';
import { useFrame } from '@react-three/fiber';
import * as THREE from 'three';
import { $scrollProgress, $sceneReady } from '@/stores';

gsap.registerPlugin(ScrollTrigger, useGSAP);
ScrollTrigger.config({ ignoreMobileResize: true });
```

Import from sub-paths where possible; GSAP core barrel is fine (no Vite 504 issue with GSAP).

### Task 3 — Add scroll-driven camera hook inside GameSetupScene

Inside the `GameSetupScene` component, add a mutable progress ref and `useGSAP` hook:

```tsx
// Plain JS ref — GSAP writes this, useFrame reads it (Pitfall §9 single-writer rule)
const cameraProgress = useRef({ value: 0 });
const isScrollActiveRef = useRef(false);

useGSAP(() => {
  // Reduced-motion guard
  if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) return;

  const tween = gsap.to(cameraProgress.current, {
    value: 1,
    ease: 'power2.inOut',
    scrollTrigger: {
      trigger: '#scene-scroll-pin',
      start: 'top top',
      end: 'bottom top',
      scrub: true,
      onUpdate: (self) => {
        $scrollProgress.set(self.progress);
        isScrollActiveRef.current = true;
        invalidate(); // demand-mode: render this frame
      },
      onLeave: () => { isScrollActiveRef.current = false; },
      onLeaveBack: () => { isScrollActiveRef.current = false; },
    },
  });

  return () => {
    tween.scrollTrigger?.kill();
  };
}, { dependencies: [] });
```

`invalidate` is already destructured from `useThree()` in the component.

### Task 4 — Add useFrame to apply camera interpolation

Inside `GameSetupScene`, add a `useFrame` that reads `cameraProgress.current.value` and lerps the camera:

```tsx
const startPos = new THREE.Vector3(...CAMERA_POSITION);
const endPos = new THREE.Vector3(...CAMERA_END_POSITION);
const startTarget = new THREE.Vector3(...CAMERA_TARGET);
const endTarget = new THREE.Vector3(...CAMERA_END_TARGET);
const _tmpPos = new THREE.Vector3();
const _tmpTarget = new THREE.Vector3();

useFrame(({ camera }) => {
  const t = cameraProgress.current.value;
  if (t === 0) return; // no-op at rest — avoids redundant writes at 0%

  _tmpPos.lerpVectors(startPos, endPos, t);
  _tmpTarget.lerpVectors(startTarget, endTarget, t);

  camera.position.copy(_tmpPos);
  camera.lookAt(_tmpTarget);
});
```

Allocate `THREE.Vector3` instances outside `useFrame` (avoid per-frame GC pressure).
